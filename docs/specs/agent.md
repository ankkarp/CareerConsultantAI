### Agent / Orchestrator (Оркестрация)

**Модуль:** `model` (основная LLM логика), `tg-module/bot.py`

| Параметр                            | Спецификация                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Шаги**                          | 1. Пользователь отправляет сообщение в Telegram. 2.`tg-module/bot.py` вызывает `POST /start_talk/` модуля `model` с `user_id` и `prompt`. 3. Модуль `model` (внутренняя логика) формирует ответ (возможно, с вызовом `Retriever` или `web_search`). 4. Ответ возвращается в бот и отправляется пользователю. |
| **Правила переходов** | Определяются логикой внутри модуля `model` (не раскрыта). Telegram бот обрабатывает команды: `/start`, `/help`, `/cancel` — и возвращает пользователя в начальное состояние.                                                                                                                                                                                    |
| **Stop Condition**                    | Команда `/cancel` в Telegram боте. Для агента — естественное завершение диалога или явная остановка.                                                                                                                                                                                                                                                                                                        |
| **Retry / Fallback**                  | При ошибке вызова LLM API в `tg-module/llm_client.py` нет явной логики повторных попыток. Ошибка логируется, пользователю отправляется сообщение об ошибке.                                                                                                                                                                                                              |
