
### Memory / Context (Управление состоянием)

**Модули:** `tg-module`, `model`

| Параметр         | Спецификация                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Session State**  | Управляется на уровне Telegram-бота через `aiogram` (FSM — Finite State Machine) или хранение в памяти. `user_id` передается в каждый запрос к LLM API.                                                                                                                                                                                                                 |
| **Memory Policy**  | Контекст диалога удерживается внутри LLM API. История сообщений сохраняет в Sqlite базу данный, в кеше хранится информация о текущем состоянии рекомендаций по профессиям и курсам для пользователя в текущем диалоге.
| **Context Budget** | Ограничено максимальным размером контекстного окна используемой LLM. Управление (обрезание истории) реализовано на стороне модуля `model`. Для `professions_vector_index` бюджет ограничен размером встраиваемого текста профессии. |
