
### Memory / Context (Управление состоянием)

**Модули:** `tg-module`, `model`

| Параметр         | Спецификация                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Session State**  | Управляется на уровне Telegram-бота через `aiogram` (FSM — Finite State Machine) или хранение в памяти. `user_id` передается в каждый запрос к LLM API.                                                                                                                                                                                                                 |
| **Memory Policy**  | Не явно специфицирована. По-видимому, контекст диалога удерживается внутри LLM API (модуль `model`) на основе `user_id`. История сообщений не сохраняется между перезапусками сервиса (in-memory).                                                                                                         |
| **Context Budget** | Ограничено максимальным размером контекстного окна используемой LLM. Управление (обрезание истории) реализовано на стороне модуля `model`. Для `professions_vector_index` бюджет ограничен размером встраиваемого текста профессии (не определен жестко). |
